REST API(Representational state transfer)
 - 네트워크 구조의 한 형식
 - 서버의 자원을 정의
 - 자원에 대한 주소를 지정하는 방법
 - 주소 외에도 http 요청 메서드 사용
  - GET : 서버 자원을 가져올 때 사용. 요청의 본문(body)에 데이터를 넣지 않음. 데이터를 서버로 보낼 때는 쿼리 스트링 사용
  - POST : 서버에 자원을 등록할 때 사용. 요청의 본문에 등록할 데이터를 보냄
  - PUT : 서버의 자원을 요청에 들어 있는 자원으로 치환하려고 할 때 사용. 요청의 본문에 치환할 데이터를 넣어 보냄
  - PATCH : 서버 자원의 일부만 수정하고 할 때 사용합니다. 요청의 본문에 수정할 데이터를 넣어 보냄
  - DELETE : 서버의 자원을 삭제하고자 할 때 사용.
- 주소 하나가 여러개의 요청 메서드를 가질 수 있음
 - /user : GET은 조회 POST는 등록으로 처리 가능
- GET메서드는 브라우저에서 캐싱할 수도 있어서 같은 주소의 GET 요청을 할 때 서버가 아니라 캐시에서 가져올 수 있음. 성능 향상됨
- http 프로토콜을 사용하면 클라이언트가 누구든 상관 없이 서버와 소통할 수 있다. 서버와 클라가 분리된 것이다.
