    - 노드 패키지 버전은 항상 세 자리로 이루어져 있다. 노드의 버전도 세자리다.
    - SemVer( Semantic versioning ) 방식의 버전 넘버링을 따르기 때문이다.

버전 자리 수의 의미
    - 첫번째
         - major버전
         - 0이면 초기 개발 중인 것이고 1부터는 정식 버전
         - 하위 호환이 안될 정도의 패키지 수정이 된 경우 업데이트
    - 두번째
        - minor버전
        - 하위 호환이 되는 기능 업데이트
    - 세번째
        - patch 버전
        - 기존 기능에 문제가 있는 경우 수정 배포 버전

package.json에 붙는 기호의 의미
    - ^
        - minor 버전 까지만 설치 또는 업데이트 한다.
    - ~
        - patch 버전 까지만 설치 또는 업데이트 한다.
    - > < >= <= = 
        - 등호 그대로의 의미
    - @latest
        - 항상 최신 버전을 설치
        - x로도 표시 가능
        - npm i express@latest/express@x로 작성
